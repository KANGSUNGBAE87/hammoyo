브라우저 저장 앱 화면

현재 앱 화면 범위

이 문서는 GitHub Pages에서 제공되는 함모여 앱 화면에 적용됩니다. 현재 화면은 Toss login 계정 동기화 전까지 브라우저 localStorage를 우선 사용하고, 일반 공유 링크에는 방 제목과 후보 일정 snapshot이 포함됩니다.

처리하는 데이터

  • 방 제목
  • 후보 날짜, 시간, 장소 또는 메모
  • 참여자가 입력한 이름 또는 별명
  • 후보 시간별 선호 응답

저장 위치와 보관 기간

위 데이터는 사용자의 현재 브라우저 localStorage에 저장됩니다. 일반 공유 링크를 직접 복사해 다른 사람에게 보내면 링크 안의 방 제목, 후보 날짜/시간, 장소/메모 snapshot을 링크 수신자가 볼 수 있습니다. 응답 데이터는 Toss login/account 동기화 전에는 방장 기기로 자동 동기화되지 않습니다.

삭제 방법

함모여 화면의 개인정보 메뉴에서 로컬 데이터 지우기를 누르면 현재 브라우저에 저장된 데이터가 삭제됩니다.

현재 GitHub Pages 화면에는 계정 생성 UX가 없어서 계정 삭제 요청 대상이 없습니다. 향후 Toss login, Supabase, Google Play 계정 흐름을 연결하면 앱 안의 로컬 삭제와 별도의 계정/서버 데이터 삭제 요청 경로를 제공해야 합니다.

제3자 제공과 외부 전송

현재 GitHub Pages 화면은 사용자의 응답을 외부 서버나 AI 모델로 직접 전송하지 않습니다. 단, 사용자가 일반 공유 링크를 복사해 전달하면 링크 안의 공개 후보 정보가 링크 수신자에게 제공됩니다. Supabase/DeepSeek 서버 함수는 별도 backend 경계로 연결되어 있으며, Toss login/account 동기화 UI를 켤 때 이 방침과 앱 내 고지를 업데이트해야 합니다.

문의

정식 출시 전 문의는 프로젝트 관리자에게 전달해 주세요. Google Play 또는 Apps in Toss 제출 전에는 공개 문의 이메일, 외부 개인정보처리방침 URL, 데이터 삭제 요청 URL을 확정해야 합니다.

제출 전 확정 필요: 공개 문의 이메일 미정 · 외부 개인정보처리방침 URL 미정 · 데이터 삭제 요청 URL 미정

제출 전 경로 점검: 문의 경로 · 데이터 삭제 요청 경로

English Summary

The current GitHub Pages app stores room data in this browser first. General share links include a public snapshot of the room title and candidate times so recipients can open the response screen. Responses are not automatically synced back to the host before Toss login/account sync is enabled.

You can remove local browser data from the Privacy screen by choosing Clear local data. There is no account deletion flow yet because this GitHub Pages screen does not expose account creation. Public contact email, external privacy-policy URL, and data-deletion request URL must be finalized before Apps in Toss or Google Play submission.
