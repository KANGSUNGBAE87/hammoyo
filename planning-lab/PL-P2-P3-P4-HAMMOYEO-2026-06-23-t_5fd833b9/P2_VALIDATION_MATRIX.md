# P2 Validation Matrix

| 질문 | 현재 판단 | 근거 | 반대 근거 | P3 검증 필요성 |
|---|---|---|---|---|
| Primary User가 충분히 좁은가 | [FACT] 방장/주최자/총무/운영진으로 좁힘 | P2/P3/프로젝트 브리프 공통 | 참여자는 secondary라 입력 경험이 무너지면 전체 가치도 무너짐 | 높음 |
| 핵심 use situation이 좁은가 | [INFERENCE] no-unanimous/conflict 직후 | direct substitutes가 generic scheduling을 덮음 | 실제로 이 상황이 충분히 자주 있는지 미검증 | 높음 |
| 가치 제안이 대체재와 다른가 | [INFERENCE] fallback 추천 + 관계 안전 문구 | prior P3/pre-P4 synthesis | 추천 신뢰가 없으면 차별점 약함 | 높음 |
| 수익화가 말이 되는가 | [UNKNOWN] 아직 WTP 없음 | live validation 0 responses | 반복 세그먼트가 약하면 유료 불가 | 매우 높음 |
| P3로 갈 가치가 있는가 | [FACT] 기존 P2는 AI_GRADE_B / AI_RECOMMEND_PROMOTE | 공식 P2 report | 이후 P3/pre-P4는 검증 부족을 지적 | 이미 P3 수행됨 |
